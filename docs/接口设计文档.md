## 2. 统一响应格式

### 2.1 成功响应

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { }
}
```

### 2.2 错误响应

```json
{
  "code": 400,
  "message": "错误描述",
  "data": null
}
```

### 2.3 HTTP 状态码对照

| HTTP 状态码 | 业务码 | 说明 |
|-------------|--------|------|
| 200 | 200 | 操作成功 |
| 400 | 400 | 请求参数错误 / 表单校验失败 |
| 401 | 401 | 未认证 / Token 无效或过期 |
| 405 | 405 | 不支持的请求方法 |
| 500 | 500 | 系统异常 |

## 3. 认证机制

### 3.1 登录获取 Token

```
POST /login
```

**请求体：**
```json
{
  "username": "admin",
  "password": "123456"
}
```

**成功响应：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "token": "eyJhbGciOiJIUzM4NCJ9...",
    "username": "系统管理员"
  }
}
```

**JWT Token 信息：**
- 签名算法：HS384
- 有效期：2 小时 (7200000ms)
- 包含字段：`id`, `username`

### 3.2 请求头携带 Token

所有需要认证的接口（除 `/login` 外）必须在 Header 中携带 Token：

```
token: eyJhbGciOiJIUzM4NCJ9...
```

### 3.3 Token 失效处理

当 Token 无效或过期时，后端返回 HTTP 401，前端自动跳转到登录页。

| 参数 | 位置 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| file | form-data | MultipartFile | 是 | 图片文件 |

**请求方式：** `multipart/form-data`

**文件限制：**
| 限制项 | 值 |
|--------|-----|
| 最大体积 | 5MB |
| 允许格式 | .jpg, .jpeg, .png, .gif, .bmp, .webp |

**处理流程：**
1. 校验文件非空、大小 ≤ 5MB、扩展名在允许列表中
2. 生成唯一文件名：`UUID + 扩展名`
3. MinIO 上传：`minioClient.putObject(bucket, fileName, inputStream, size, contentType)`
4. 返回文件访问 URL

**响应 data：**
```json
"https://objectstorageapi.hzh.sealos.run/e09w3hj8-system/a1b2c3d4-xxxx.jpg"
```

| 字段 | 类型 | 说明 |
|------|------|------|
| data | String | 文件访问 URL（公开可读） |
