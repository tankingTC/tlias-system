## 3. 表结构详细设计

### 3.1 部门表 (dept)

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INT | PK, AUTO_INCREMENT | - | 主键，部门ID |
| name | VARCHAR(50) | NOT NULL | - | 部门名称 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：**
- `PRIMARY KEY (id)`

**说明：** 部门表为基础数据表，存储组织架构信息。

---

### 3.2 员工表 (emp)

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INT | PK, AUTO_INCREMENT | - | 主键，员工ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | - | 用户名（登录账号） |
| password | VARCHAR(100) | NOT NULL | - | 密码（BCrypt加密存储） |
| name | VARCHAR(50) | NOT NULL | - | 姓名 |
| gender | TINYINT | NOT NULL | 0 | 性别（0-女，1-男） |
| phone | VARCHAR(20) | NULLABLE | NULL | 手机号 |
| job | VARCHAR(50) | NULLABLE | NULL | 职位 |
| dept_id | INT | NULLABLE, FK→dept.id | NULL | 所属部门ID |
| image | VARCHAR(255) | NULLABLE | NULL | 头像URL（对象存储地址） |
| entry_date | DATE | NULLABLE | NULL | 入职日期 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：**
- `PRIMARY KEY (id)`
- `UNIQUE KEY uk_username (username)`
- `KEY idx_dept_id (dept_id)`

**外键关系：**
- `dept_id` → `dept.id`（逻辑外键，未建立物理外键约束）

**说明：** 员工表同时作为系统用户表，username 用于登录认证。password 字段通过 `@JsonIgnore` 注解防止序列化泄露。

---

### 3.3 班级表 (class)

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INT | PK, AUTO_INCREMENT | - | 主键，班级ID |
| name | VARCHAR(100) | NOT NULL | - | 班级名称 |
| teacher_id | INT | NULLABLE, FK→emp.id | NULL | 班主任ID |
| start_date | DATE | NULLABLE | NULL | 开班时间 |
| end_date | DATE | NULLABLE | NULL | 结课时间 |
| status | TINYINT | NOT NULL | 0 | 班级状态（0-未开班，1-开班中，2-已结课） |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：**
- `PRIMARY KEY (id)`
- `KEY idx_teacher_id (teacher_id)`

**外键关系：**
- `teacher_id` → `emp.id`（逻辑外键）

**状态枚举：**
| 值 | 说明 |
|----|------|
| 0 | 未开班 |
| 1 | 开班中 |
| 2 | 已结课 |

### 3.7 反馈表 (feedback)

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INT | PK, AUTO_INCREMENT | - | 主键，反馈ID |
| type | TINYINT | NOT NULL | 0 | 反馈类型（0-学员反馈，1-员工反馈） |
| title | VARCHAR(200) | NOT NULL | - | 反馈标题 |
| content | TEXT | NOT NULL | - | 反馈内容 |
| contact | VARCHAR(50) | NULLABLE | NULL | 联系人 |
| phone | VARCHAR(20) | NULLABLE | NULL | 联系电话 |

| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

|------|----|------|
| type | 0 | 学员反馈 |
| type | 1 | 员工反馈 |
| status | 0 | 未处理 |

| status | 2 | 已处理 |

---

### 3.8 操作日志表 (operation_log)

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INT | PK, AUTO_INCREMENT | - | 主键 |
| username | VARCHAR(50) | NULLABLE | NULL | 操作人 |
| method | VARCHAR(20) | NULLABLE | NULL | 请求方法（POST/PUT/DELETE） |
| url | VARCHAR(255) | NULLABLE | NULL | 请求路径 |
| params | TEXT | NULLABLE | NULL | 请求参数（密码字段自动脱敏） |
| ip | VARCHAR(50) | NULLABLE | NULL | IP地址 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 操作时间 |

**索引：**
- `PRIMARY KEY (id)`
- `KEY idx_username (username)`
- `KEY idx_create_time (create_time)`

**说明：** 由 `OperationLogAspect` AOP 切面自动记录，仅记录 POST/PUT/DELETE 操作。日志记录失败不影响业务执行。

## 4. 表关系汇总

| 关系 | 父表 | 子表 | 关联字段 | 关系类型 |
|------|------|------|----------|----------|
| 部门-员工 | dept | emp | emp.dept_id → dept.id | 1:N |
| 员工-班级 | emp | class | class.teacher_id → emp.id | 1:N |
| 班级-学员 | class | student | student.class_id → class.id | 1:N |
| 班级-考试 | class | exam | exam.class_id → class.id | 1:N |
| 学员-就业 | student | job | job.student_id → student.id | 1:1 |
| 员工-工作经历 | emp | work_experience | work_experience.emp_id → emp.id | 1:N |
